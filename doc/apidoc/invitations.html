GET /v1/users/:user_id/invitations/:id
Retrieve information about a meeting or invitation if they have an invitation

Errors

Code Description
404 The invitation was not found

Examples

GET /v1/users/1/invitations/1
200
{
  "invitation": {
    "meeting_link": "http://test.com/v1/meetings/1/NCQ3JDJkMjUxNjhjLTMwOTQtNDc2YS05ZWIzLTIwODUxN2M5YTg3MA==",
    "links": {
      "self": "/v1/users/1/invitations/1"
    },
    "meeting": {
      "id": 1,
      "place": "Columbia, The angelic Monument Tower",
      "date": "2017-05-05",
      "time": "10:00",
      "links": {
        "self": "/v1/users/1/meetings/1"
      }
    }
  }
}

GET /v1/users/:user_id/invitations/:id/status
Check the status of an invitation they received

Errors

Code Description
404 The invitation was not found

Metadata

:status:
- pending
- accepted
- declined

Examples

GET /v1/users/1/invitations/1/status
200
{
  "invitation": {
    "status": "accepted"
  }
}
GET /v1/users/1/invitations/-1/status
404
>>>

PUT /v1/users/:user_id/invitations/:id/execute
Take action on the invitation

Errors

Code Description
400 The invitation has validation errors
404 The invitation was not found

Examples

PUT /v1/users/1/invitations/1/execute
{
  "invitation": {
    "action": "accept"
  }
}
200
{
  "invitation": {
    "status": "accepted"
  }
}
PUT /v1/users/1/invitations/1/execute
{
  "invitation": {
    "action": ""
  }
}
400
{
  "errors": {
    "action": [
      "can't be blank"
    ]
  }
}
PUT /v1/users/1/invitations/-1/execute
404

Params

Param name Description
invitation
required

Invitation info

Validations:

  • Must be a Hash
invitation[action]
required

Action to be executed

Validations:

  • Must be a String


Metadata: 
- accept
- decline
